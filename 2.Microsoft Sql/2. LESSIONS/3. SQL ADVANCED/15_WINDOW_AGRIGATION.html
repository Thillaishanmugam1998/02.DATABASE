SECTION 01 Normal Aggregate Function
Question: What is the average salary of ALL employees? — A normal aggregate collapses all rows into one single value.
SQL — Normal Aggregate 
SELECT
    AVG(Salary) AS CompanyAverageSalary
FROM Employees;

-- Returns: ONE row — entire table collapsed to a single number
⟳ Scanning all 7 rows...
1Scan all rows
→
2Sum salaries
→
3Count rows = 7
→
4395000 ÷ 7
OUTPUT — 1 row returned
CompanyAverageSalary
56,428
✗ What you LOSE
Employee names, departments, individual salaries — all gone. 7 rows → 1 row.
✓ What you GET
One clean overall number. Perfect for a company-wide KPI dashboard tile.
📊 Output: 1 row · 7 rows collapsed · No detail retained
Key Point: Normal aggregate always returns exactly one row. All individual employee data is permanently lost in the result.
SECTION 02 GROUP BY Aggregate
Question: What is the average salary per department? — GROUP BY reduces rows to one row per group.
SQL — GROUP BY Aggregate 
SELECT
    Department,
    AVG(Salary) AS DepartmentAverageSalary
FROM Employees
GROUP BY Department;

-- Returns: ONE row per department — employee-level data LOST
⟳ Grouping rows by Department...

2Group by Dept

3AVG per group

4Emit 1 row/group

Key Point: GROUP BY always reduces rows. You get one row per unique group value. Individual row data is discarded — you cannot retrieve Alice's name from the GROUP BY result.
SECTION 03 Window Aggregate Function
Question: Show each employee with their department average salary alongside — Window functions keep all rows and add analytics per row.
SQL — Window AVG 
SELECT
    EmployeeName,
    Department,
    Salary,
    AVG(Salary) OVER (PARTITION BY Department)
        AS DepartmentAverageSalary
FROM Employees;

-- Returns: ALL 7 rows — each gets the dept average attached
SQL — Window SUM 
SELECT
    EmployeeName,
    Department,
    Salary,
    SUM(Salary) OVER (PARTITION BY Department)
        AS TotalSalaryPerDept
FROM Employees;

-- Returns: ALL 7 rows — each gets the dept total attached
SQL — Window COUNT 
SELECT
    EmployeeName,
    Department,
    COUNT(*) OVER (PARTITION BY Department)
        AS EmployeesInDept
FROM Employees;

-- Returns: ALL 7 rows — each knows the dept headcount

01 GETDATE & Date Values
GETDATE Current System Timestamp Getdate
GETDATE() returns the current database server date and time as a DATETIME value. It takes no arguments. Use it for logging event timestamps, calculating ages, or comparing records against "right now." It refreshes each time the query runs.
GETDATE()
Now
Hard-coded
SELECT OrderID, CreationTime,
  '2025-08-20' AS HardCoded,
  GETDATE() AS Today
FROM Sales.Orders;
02 Date Part Extractions
DATETRUNC Truncate to Unit Extract
DATETRUNC zeroes out every component smaller than the specified unit, returning a clean boundary date. For example, truncating 2025-08-20 14:35:22 to year gives 2025-01-01 00:00:00. Ideal for GROUP BY aggregations.
DATETRUNC( datepart, date )
Date
Unit →
SELECT
  DATETRUNC(year, CreationTime) AS Year_dt,
  DATETRUNC(day, CreationTime) AS Day_dt,
  DATETRUNC(minute, CreationTime) AS Minute_dt
FROM Sales.Orders;
DATENAME Named String Part Extract
DATENAME returns the name of the specified date part as a VARCHAR string — perfect when you need "January" instead of 1, or "Wednesday" instead of 4. Great for human-readable reports.
DATENAME( datepart, date )
Date
Part →
SELECT
  DATENAME(month, CreationTime) AS Month_dn,
  DATENAME(weekday, CreationTime) AS Weekday_dn
FROM Sales.Orders;
DATEPART Numeric Part Extraction Extract
DATEPART returns the specified component of a date as an INTEGER. Unlike DATENAME it gives numbers, making it suitable for arithmetic and filtering (e.g., WHERE DATEPART(month,...) = 2).
DATEPART( datepart, date )
Date
SELECT
  DATEPART(year, CreationTime) AS Year_dp,
  DATEPART(month, CreationTime) AS Month_dp,
  DATEPART(quarter, CreationTime) AS Quarter_dp
FROM Sales.Orders;
YEAR / MONTH / DAY Shorthand Extractors Extract
YEAR, MONTH, and DAY are convenient shorthand functions equivalent to DATEPART(year,…), DATEPART(month,…), and DATEPART(day,…). They always return integers and are the most readable way to extract the big three parts.
YEAR(date) | MONTH(date) | DAY(date)

  MONTH(OrderDate) AS OrderMonth,

FROM Sales.Orders;
03 EOMONTH
EOMONTH End of Month Date Eomonth
EOMONTH returns the last day of the month for any given date as a DATE value. An optional second argument offsets by N months. Useful for billing cycles, SLA deadlines, and monthly reporting boundaries.

Date Offset

04 FORMAT
FORMAT Date String Formatting Format
FORMAT converts a date/number into a formatted string using .NET format specifiers. It gives you the most flexibility for display but is slower than CONVERT for large datasets. Use for reports and front-end display queries.
FORMAT( value, format_string )
Date
SELECT
  FORMAT(CreationTime, 'MM-dd-yyyy') AS USA_Format,
  FORMAT(CreationTime, 'dd-MM-yyyy') AS EURO_Format,
  FORMAT(CreationTime, 'dddd') AS DayName,
  FORMAT(CreationTime, 'MMMM') AS MonthName
FROM Sales.Orders;
05 CONVERT
CONVERT Type & Style Conversion Convert
CONVERT changes a value from one data type to another. A third optional argument (style code) controls date formatting — e.g., style 32 gives US format, 34 gives European. Preferred over FORMAT for high-volume queries due to better performance.
CONVERT( target_type, value [, style] )
DateTime
ExpressionResult
SELECT
  CONVERT(INT, '123'),
  CONVERT(DATE, CreationTime),
  CONVERT(VARCHAR, CreationTime, 32) AS USA,
  CONVERT(VARCHAR, CreationTime, 34) AS EURO
FROM Sales.Orders;
06 CAST
CAST Standard Type Casting Cast
CAST is the ANSI SQL standard for type conversion. Unlike CONVERT, it does not support a style code — just source and target type. Use CAST when you want portable code that works across SQL dialects. Prefer it over CONVERT when no date-style formatting is needed.
CAST( value AS target_type )
DateTime
ExpressionResult
SELECT
  CAST('123' AS INT),
  CAST(123 AS VARCHAR),
  CAST(CreationTime AS DATE)
FROM Sales.Orders;
07 DATEADD & DATEDIFF
DATEADD Add / Subtract Time Date Math
DATEADD adds (or subtracts with negative values) a number of date units to a date, returning a new DATETIME. Use it to calculate due dates, expiry windows, or offset delivery schedules by days, months, or years.
DATEADD( datepart, number, date )
Date
Unit N
Result
SELECT
  DATEADD(day, -10, OrderDate) AS TenDaysBefore,
  DATEADD(month, 3, OrderDate) AS ThreeMonthsLater
FROM Sales.Orders;
DATEDIFF Difference Between Dates Date Math
DATEDIFF calculates the number of specified date units between two dates (end minus start). It counts boundary crossings, so from Dec 31 to Jan 1 = 1 year regardless of hours. Widely used for age calculation, shipping duration, and time-gap analysis.

08 ISDATE
ISDATE Date Validation Isdate
ISDATE checks whether a string or expression is a valid date/time value. Returns 1 if valid, 0 if not. Combine with a CASE expression to substitute a fallback date for invalid rows — critical for safely importing messy external data.
ISDATE( expression ) → 1 | 0
Add value
SELECT OrderDate,
  ISDATE(OrderDate) AS IsValidDate,
  CASE WHEN ISDATE(OrderDate) = 1
    THEN CAST(OrderDate AS DATE)
    ELSE '9999-01-01'
  END AS NewOrderDate
FROM (...) AS t;

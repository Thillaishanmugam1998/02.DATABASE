01
Data Aggregation with COALESCE
AVG(Score) ignores NULL — Replace NULL with 0 to include it

Problem: NULL in AVG() Calculation

CustomerID FirstName Score
1 John 85
2 Sarah NULL
3 Mike 92
4 Emma NULL
SELECT AVG(Score) AS AvgScore
FROM Customers;
⚠️ Result Without COALESCE
AvgScore = 88.5
Only 85 and 92 are averaged (NULL ignored)
(85 + 92) / 2 = 88.5

Solution: Replace NULL with COALESCE

SELECT AVG(COALESCE(Score, 0)) AS AvgScore
FROM Customers;
Original Score COALESCE(Score, 0)
85 85
NULL 0
92 92
NULL 0
✓ Result With COALESCE
AvgScore = 67.25
All 4 values included
(85 + 0 + 92 + 0) / 4 = 67.25
💡 Key Insight: NULL is treated as "absence of data" by aggregate functions. Use COALESCE to convert NULL to a meaningful default before aggregation.
02
Mathematical Operators with NULL
NULL + anything = NULL — Use COALESCE for arithmetic

Challenge: String Concatenation

CustomerID FirstName LastName
1 John Smith
2 Sarah NULL
3 Mike Johnson
SELECT FirstName + ' ' + LastName AS FullName
FROM Customers;
⚠️ Without COALESCE
✓ John Smith
✗ NULL (Sarah + ' ' + NULL = NULL)
✓ Mike Johnson

Solution: COALESCE with Empty String

SELECT FirstName + ' ' + COALESCE(LastName, '') AS FullName
FROM Customers;
FirstName COALESCE(LastName, '') FullName
John Smith John Smith
Sarah '' Sarah
Mike Johnson Mike Johnson

Bonus Points: Arithmetic with Scores

SELECT Score, COALESCE(Score, 0) + 10 AS ScoreWithBonus
FROM Customers;
Score COALESCE(Score, 0) + 10
85 85 95
NULL 0 10
💡 Golden Rule: NULL propagates through all arithmetic and string operations. Always use COALESCE to convert NULL before math.
03
Sorting with NULL Control
ORDER BY — Force NULLs to appear last (or first)

Problem: Unpredictable NULL Placement

CustomerID Score
1 85
2 NULL
3 92
4 NULL
SELECT * FROM Customers
ORDER BY Score ASC;
⚠️ Database Default (Unpredictable)
NULL (Customer 2) — Position varies by database!
NULL (Customer 4)
85 (Customer 1)
92 (Customer 3)

Solution: Explicit CASE Control

SELECT * FROM Customers
ORDER BY CASE WHEN Score IS NULL THEN 1 ELSE 0 END, Score;
🔍 How it works:
1
Step 1
CASE returns 0 for non-NULL, 1 for NULL
2
Step 2
0s sort before 1s → non-NULL rows first
3
Step 3
Then sort by Score → NULLs guaranteed last!
✓ Controlled Sort Order
85 (Customer 1) ✓
92 (Customer 3) ✓
NULL (Customer 2) ← Last!
NULL (Customer 4) ← Last!
💡 Pro Tip: Change the CASE logic to put NULLs first: CASE WHEN Score IS NULL THEN 0 ELSE 1 END
04
NULLIF – Prevent Division by Zero
NULLIF(Quantity, 0) converts 0 to NULL to avoid errors

The Problem: Division by Zero

OrderID Sales Quantity
1 100 5
2 200 0
3 150 3
SELECT Sales / Quantity AS Price
FROM Orders;
❌ Error on Row 2
Msg 8134: Divide by zero error encountered
Query fails completely!

Solution: NULLIF (Conditional NULL)

SELECT Sales / NULLIF(Quantity, 0) AS Price
FROM Orders;
Sales Quantity NULLIF(Qty, 0) Result
100 5 5 20
200 0 NULL NULL
150 3 3 50
✓ Query Succeeds
20 (Order 1)
NULL (Order 2) ← No error!
50 (Order 3)
💡 NULLIF Logic: NULLIF(expr1, expr2) returns NULL if expr1 = expr2, otherwise returns expr1. Perfect for converting zero divisors to NULL.
05
IS NULL / IS NOT NULL – Filtering
Never use = NULL or ≠ NULL — Always use IS NULL

Finding Customers Without Scores

CustomerID Name Score
1 John 85
2 Sarah NULL
3 Mike 92
4 Emma NULL

❌ Wrong: WHERE Score = NULL

No Results
NULL comparisons always return unknown. This never works!

✓ Correct: WHERE Score IS NULL

2 Rows
Sarah (CID 2) and Emma (CID 4)
SELECT * FROM Customers
WHERE Score IS NULL;
✓ Find Rows with NULL
2, Sarah, NULL
4, Emma, NULL

Opposite: IS NOT NULL

SELECT * FROM Customers
WHERE Score IS NOT NULL;
✓ Find Rows with Valid Data
1, John, 85
3, Mike, 92
💡 Critical Rule: NULL is "unknown", so SQL can never confirm NULL = NULL. Always use IS NULL and IS NOT NULL for comparisons.
06
LEFT ANTI JOIN – Find Unmatched Records
Customers with no orders using NULL detection

The Pattern: Find Non-Matching Rows

Customers

C1: John ✓ (has orders)
C2: Sarah ✗ (no orders)
C3: Mike ✓ (has orders)
C4: Emma ✗ (no orders)
→

Result: Unmatched Only

C2: Sarah
C4: Emma

Query Pattern

SELECT c.* FROM Customers AS c
LEFT JOIN Orders AS o ON c.CustomerID = o.CustomerID
WHERE o.CustomerID IS NULL;
Step What Happens
1. LEFT JOIN Keep ALL customers, add matching orders (NULL if no match)
2. WHERE ... IS NULL Filter only rows where OrderID is NULL (unmatched)
3. Result Customers with NO orders!
✓ Customers Without Orders
2, Sarah
4, Emma
💡 LEFT ANTI JOIN Pattern: LEFT JOIN + IS NULL finds all rows from left table that have NO match in right table. Perfect for "find what's missing" queries.
07
NULL vs Empty String vs Spaces
Three different values, three different behaviors

Understanding the Difference

NULL

NULL
Unknown/Missing data. No value at all. Can't be compared.

Empty String

''
A real string with zero characters. A legitimate value.

Spaces

' '
A string containing space characters. Visible content.

Example Data

ID Category DATALENGTH Meaning
1 'A' 1 1 character
2 NULL NULL No data to measure
3 '' 0 Empty, but real value
4 ' ' 2 2 space characters

Handling All Three: TRIM + NULLIF + COALESCE

SELECT
  Category,
  TRIM(Category) AS Trimmed,
  NULLIF(TRIM(Category), '') AS NullIfEmpty,
  COALESCE(NULLIF(TRIM(Category), ''), 'unknown') AS Final
FROM Orders;
Original TRIM() NULLIF(TRIM(), '') COALESCE(..., 'unknown')
'A' 'A' 'A' 'A'
NULL NULL NULL 'unknown'
'' '' NULL 'unknown'
' ' '' NULL 'unknown'
✓ The Processing Pipeline
TRIM: Remove leading/trailing spaces
NULLIF: Convert empty strings to NULL
COALESCE: Replace NULL with 'unknown' default

All three cases (NULL, empty, spaces) end up as 'unknown'!
💡 Data Cleanup Pattern: Use TRIM + NULLIF + COALESCE as a triple combo to normalize all "empty-like" values consistently.

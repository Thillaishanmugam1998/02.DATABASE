01 CTE Fundamentals

Common Table Expressions (CTEs) are temporary named result sets that exist only within a single query. They improve code readability and allow you to break complex queries into simpler parts. CTEs are defined with the WITH keyword and can be referenced in the main SELECT statement.
SQL
WITH CTE_Example AS ( SELECT OrderID, CustomerID, Sales FROM Sales.Orders WHERE Sales > 100 ) SELECT * FROM CTE_Example;

02 The ORDER BY Rule

ORDER BY is NOT allowed inside a CTE definition unless you use TOP, OFFSET, or FOR XML. This rule exists because CTEs are temporary result sets, and the order of rows should be determined by the outer query, not the CTE itself. Always place ORDER BY in the main query.
❌ WRONG
SQL
WITH CTE_Test AS ( SELECT * FROM Sales.Orders ORDER BY Sales DESC ) SELECT * FROM CTE_Test;
✓ CORRECT
SQL
WITH CTE_Test AS ( SELECT * FROM Sales.Orders ) SELECT * FROM CTE_Test ORDER BY Sales DESC;

03 The Problem Without CTEs

Without CTEs, you must write the same query logic multiple times. This leads to code duplication, maintenance issues, and reduced readability. Imagine needing the same filtered result in five different places in your code!

04 Stand-Alone CTE Solution

A Stand-Alone CTE is a single, independent CTE used once in a query. It replaces the need to write the logic inline in the SELECT statement. This improves readability and makes the query self-documenting.
SQL
WITH CTE_TotalSales AS ( SELECT CustomerID, SUM(Sales) AS TotalSales FROM Sales.Orders GROUP BY CustomerID ) SELECT CustomerID, TotalSales FROM CTE_TotalSales;

05 Nested CTEs (Multiple CTEs)

Nested CTEs allow one CTE to reference another CTE. This is powerful for breaking down complex logic into smaller, logical steps. Build one CTE, then use it as input to the next CTE, creating a chain of transformations.
SQL
WITH CTE_TotalSales AS ( SELECT CustomerID, SUM(Sales) AS TotalSales FROM Sales.Orders GROUP BY CustomerID ), CTE_CustomerRank AS ( SELECT CustomerID, TotalSales, RANK() OVER (ORDER BY TotalSales DESC) AS Rank FROM CTE_TotalSales ) SELECT * FROM CTE_CustomerRank;

06 Recursive CTEs

Recursive CTEs allow a CTE to reference itself, enabling you to generate sequences, number ranges, or traverse hierarchical data (like employee-manager relationships). They consist of an anchor member (initial query) and a recursive member (self-reference).
SQL
WITH CTE_NumberSeries AS ( -- Anchor Member: Start with 1 SELECT 1 AS Number UNION ALL -- Recursive Member: Add 1 each iteration SELECT Number + 1 FROM CTE_NumberSeries WHERE Number < 10 ) SELECT * FROM CTE_NumberSeries;

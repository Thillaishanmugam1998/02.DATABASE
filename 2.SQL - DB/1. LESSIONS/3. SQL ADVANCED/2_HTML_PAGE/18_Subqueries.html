⚡ The Golden Rule of Subqueries
A subquery is a complete SQL SELECT statement nested inside another query. The inner query executes first, and its result is passed to the outer query. They can appear in SELECT, FROM, WHERE, and JOIN clauses.

SECTION 01
Subquery Result Types
Subqueries can return three types of results: a Scalar (single value — one row, one column), a Row (single row, multiple columns), or a Table (multiple rows and columns). The result type determines where and how the subquery can be used.
SQL — Scalar
-- Returns ONE value (single row, single column) SELECT AVG(Sales) -- ← scalar result FROM Sales.Orders;
SQL — Row
-- Returns ONE column (many rows) SELECT CustomerID -- ← row/list result FROM Sales.Orders;
SQL — Table
-- Returns a full table (many rows, many columns) SELECT OrderID, OrderDate -- ← table result FROM Sales.Orders;

SECTION 02
Subquery in FROM Clause
A subquery in the FROM clause acts as a derived table (inline view). The outer query treats the subquery's result as if it were a regular table. You must give it an alias using AS t. Great for pre-aggregating or pre-filtering before the main query runs.
SQL — Task 1
-- Task 1: Products with price higher than average SELECT * FROM ( -- Subquery (derived table) SELECT ProductID, Price, AVG(Price) OVER () AS AvgPrice FROM Sales.Products ) AS t WHERE Price > AvgPrice;
SQL — Task 2
-- Task 2: Rank customers by total sales SELECT *, RANK() OVER (ORDER BY TotalSales DESC) AS CustomerRank FROM ( -- Subquery aggregates first SELECT CustomerID, SUM(Sales) AS TotalSales FROM Sales.Orders GROUP BY CustomerID ) AS t;

SECTION 04
Subquery in JOIN Clause
You can use a subquery in a JOIN instead of a real table. The subquery creates a derived table that gets joined. Combined with LEFT JOIN, this lets you attach aggregated information (like TotalSales or TotalOrders) to every customer, even those with no orders.
SQL — Task 4
SELECT c.*, t.TotalSales FROM Sales.Customers AS c LEFT JOIN ( SELECT CustomerID, SUM(Sales) AS TotalSales FROM Sales.Orders GROUP BY CustomerID ) AS t ON c.CustomerID = t.CustomerID;
SQL — Task 5
SELECT c.*, o.TotalOrders FROM Sales.Customers AS c LEFT JOIN ( SELECT CustomerID, COUNT(*) AS TotalOrders FROM Sales.Orders GROUP BY CustomerID ) AS o ON c.CustomerID = o.CustomerID;

SECTION 06
Subquery with IN Operator
The IN operator checks if a value matches any value in a list returned by a subquery (row-type result). NOT IN filters out rows that match. Very useful for cross-table lookups like "orders from customers in Germany."
SQL — Task 7
SELECT * FROM Sales.Orders WHERE CustomerID IN ( SELECT CustomerID FROM Sales.Customers WHERE Country = 'Germany' );
SQL — Task 8
SELECT * FROM Sales.Orders WHERE CustomerID NOT IN ( SELECT CustomerID FROM Sales.Customers WHERE Country = 'Germany' );

SECTION 07
Subquery with ANY Operator
ANY compares a value against all values in the subquery result. salary > ANY (...) is TRUE if the salary is greater than at least one value in the list. Equivalent to > MIN(...). Useful for threshold comparisons across groups.
SQL — Task 9
-- Task 9: Female employees earning more than ANY male SELECT EmployeeID, FirstName, Salary FROM Sales.Employees WHERE Gender = 'F' AND Salary > ANY ( SELECT Salary FROM Sales.Employees WHERE Gender = 'M' );

SECTION 08
Correlated Subquery
A correlated subquery references a column from the outer query. It re-executes for every row of the outer query, passing the outer value (e.g. c.CustomerID) into the inner query. Powerful but can be slower on large datasets than a JOIN approach.
SQL — Task 10
-- Task 10: Total orders per customer (correlated) SELECT *, (SELECT COUNT(*) FROM Sales.Orders o WHERE o.CustomerID = c.CustomerID) AS TotalOrders -- ↑ references outer alias c FROM Sales.Customers AS c;

SECTION 09
Subquery with EXISTS Operator
EXISTS returns TRUE if the subquery returns at least one row. It's always a correlated subquery — the inner query references the outer table. NOT EXISTS is the inverse. EXISTS is often faster than IN because it short-circuits as soon as one match is found.
SQL — Task 11
SELECT * FROM Sales.Orders AS o WHERE EXISTS ( SELECT 1 FROM Sales.Customers AS c WHERE Country = 'Germany' AND o.CustomerID = c.CustomerID );
SQL — Task 12
SELECT * FROM Sales.Orders AS o WHERE NOT EXISTS ( SELECT 1 FROM Sales.Customers AS c WHERE Country = 'Germany' AND o.CustomerID = c.CustomerID );

Benefit
✓ Explicit handling includes all 5 rows in average calculation
❌ Silent Exclusion
AVG() silently ignores NULLs: (85+92+78)/3 = 85.00. Results hide the presence of missing data, leading to misleading insights.
✓ Explicit Treatment
COALESCE(Score, 0) converts NULLs to 0: (85+0+92+0+78)/5 = 71.00. Honest calculation that reflects incomplete data.
💡 Key Insight: SQL's AVG() function silently excludes NULL values, which can distort analysis. Use COALESCE() to explicitly handle NULLs before computing the average. This ensures your calculation reflects the actual data distribution, not just the non-NULL subset.

Advantage
✓ Jan avg = $1100 (Jan + Feb average = respects time order)
❌ No ORDER BY
AVG without ORDER BY considers all rows in partition equally, violating temporal logic. Jan's average shouldn't include future months.
✓ With ORDER BY
ORDER BY establishes row sequence. ROWS BETWEEN CURRENT ROW AND 1 FOLLOWING creates a rolling 2-row window that progresses through time correctly.
💡 Key Insight: ORDER BY is critical for time-series analysis. Without it, window aggregates treat all rows identically. Adding ORDER BY + ROWS BETWEEN clauses creates rolling/cumulative calculations that respect temporal progression. This is essential for moving averages, running totals, and trend analysis.

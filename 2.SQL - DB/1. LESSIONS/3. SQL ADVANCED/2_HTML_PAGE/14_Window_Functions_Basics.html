SECTION 05FRAME Clause — Sliding Windows
Tasks 7–11: FRAME controls which rows within a partition are included relative to the current row. A sliding window that moves row by row.
Task 7 — CURRENT ROW AND 2 FOLLOWING
SUM(Sales) OVER (
    PARTITION BY OrderStatus  ORDER BY OrderDate
    ROWS BETWEEN CURRENT ROW AND 2 FOLLOWING
) AS Total_Sales
Task 8 — 2 PRECEDING AND CURRENT ROW
SUM(Sales) OVER (
    PARTITION BY OrderStatus  ORDER BY OrderDate
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
) AS Total_Sales

    PARTITION BY OrderStatus  ORDER BY OrderDate
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW

SECTION 06Window Function Rules
Rules 1 & 2: Two critical restrictions to avoid SQL errors.
RULE 1 — Clause restrictions
INVALID — WHERE clause
WHERE
    SUM(Sales) OVER (
        PARTITION BY OrderStatus
    ) > 100;  -- ERROR!
Why it fails
Window functions run AFTER WHERE. Cannot filter on value not yet computed.
FIX — Use CTE
WITH cte AS (
    SELECT *,
        SUM(Sales) OVER (
            PARTITION BY OrderStatus
        ) AS Total_Sales
    FROM Sales.Orders
)
SELECT * FROM cte
WHERE Total_Sales > 100;
Valid clauses
SELECT and ORDER BY only.
RULE 2 — No nesting
INVALID — Nested OVER()
SUM(
    SUM(Sales) OVER (...) -- inner
) OVER (...)  -- ERROR!
Why it fails
A window function cannot be an argument of another window function.
FIX — Chain CTEs
WITH step1 AS (
    SELECT *,
        SUM(Sales) OVER (
            PARTITION BY OrderStatus
        ) AS Status_Total
    FROM Sales.Orders
)
SELECT *,
    SUM(Status_Total) OVER () AS Grand_Total
FROM step1;
Correct
Chain CTEs. Each CTE adds one level of window computation.

SECTION 07Window + GROUP BY Together

    SUM(Sales) AS Total_Sales,
    RANK() OVER (ORDER BY SUM(Sales) DESC) AS Rank_Customers
FROM Sales.Orders
GROUP BY CustomerID;

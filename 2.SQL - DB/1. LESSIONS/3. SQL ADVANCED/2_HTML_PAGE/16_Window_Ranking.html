⬡ Golden Rule — Window Function Syntax

FUNCTION_NAME() OVER (
    PARTITION BY partition_col   -- optional: split into groups
    ORDER BY sort_col ASC|DESC -- defines ordering within window
) AS alias_name
01 ROW_NUMBER()
ROW_NUMBER() assigns a unique sequential integer to each row within the window partition, starting from 1. No two rows ever share the same number — even ties get different values. Order of assignment depends entirely on the ORDER BY clause.
SQL
SELECT OrderID, ProductID, Sales, ROW_NUMBER() OVER (ORDER BY Sales DESC) AS SalesRank FROM Sales.Orders;

02 RANK()
RANK() assigns the same rank to rows with identical ORDER BY values (ties), then skips the next numbers. If two rows share rank 2, the next row gets rank 4 — creating gaps. Think Olympic medals: two silvers means no bronze.
SQL
SELECT OrderID, ProductID, Sales, ROW_NUMBER() OVER (ORDER BY Sales DESC) AS RowNum, RANK() OVER (ORDER BY Sales DESC) AS RankVal FROM Sales.Orders;

03 DENSE_RANK()
DENSE_RANK() is like RANK but never skips numbers. Tied rows still share the same rank, but the next distinct value always gets the very next integer. No gaps — hence "dense". Perfect when you need contiguous rank levels.
SQL
SELECT OrderID, ProductID, Sales, ROW_NUMBER() OVER (ORDER BY Sales DESC) AS RowNum, RANK() OVER (ORDER BY Sales DESC) AS RankVal, DENSE_RANK() OVER (ORDER BY Sales DESC) AS DenseRankVal FROM Sales.Orders;

04 PARTITION BY — Top-N per Group
PARTITION BY restarts the ranking counter for each group. Combined with ROW_NUMBER, this lets you find the top-N rows per category — e.g. highest sale per product — without a self-join. Wrap in a subquery and filter WHERE rn = 1.
SQL
-- Top-1 Sale per ProductID SELECT * FROM ( SELECT OrderID, ProductID, Sales, ROW_NUMBER() OVER ( PARTITION BY ProductID ORDER BY Sales DESC ) AS RankByProduct FROM Sales.Orders ) AS TopProductSales WHERE RankByProduct = 1;

05 NTILE(n)
NTILE(n) divides rows into n equally-sized buckets. Bucket size = total rows ÷ n (excess rows go to earlier buckets). Use it for segmentation: High/Medium/Low, quartiles, deciles. The PARTITION BY clause splits into separate tile sets per group.
SQL
-- Segment orders into High / Medium / Low sales SELECT OrderID, Sales, Buckets, CASE WHEN Buckets = 1 THEN 'High' WHEN Buckets = 2 THEN 'Medium' WHEN Buckets = 3 THEN 'Low' END AS SalesSegment FROM ( SELECT OrderID, Sales, NTILE(3) OVER (ORDER BY Sales DESC) AS Buckets FROM Sales.Orders ) AS SalesBuckets;

06 CUME_DIST()
CUME_DIST() returns the cumulative distribution: what fraction of rows have values ≤ the current row. Formula: last position of same value ÷ total rows. Returns a value between 0 (exclusive) and 1 (inclusive). Use it to find top/bottom percentile items.
SQL
-- Products in the top 40% by price SELECT Product, Price, DistRank, CONCAT(DistRank * 100, '%') AS DistRankPerc FROM ( SELECT Product, Price, CUME_DIST() OVER (ORDER BY Price DESC) AS DistRank FROM Sales.Products ) AS PriceDistribution WHERE DistRank <= 0.4;

07 PERCENT_RANK()
PERCENT_RANK() returns the relative rank as a percentage: (rank - 1) / (total rows - 1). The first row always returns 0, the last always returns 1. It uses the first occurrence of a tied value (unlike CUME_DIST which uses the last).
SQL
-- PERCENT_RANK vs CUME_DIST comparison SELECT Product, Price, CUME_DIST() OVER (ORDER BY Price DESC) AS CumeDist, PERCENT_RANK() OVER (ORDER BY Price DESC) AS PctRank FROM Sales.Products;

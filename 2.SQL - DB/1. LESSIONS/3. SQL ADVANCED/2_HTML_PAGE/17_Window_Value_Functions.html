SQL Window Functions

Window Value Functions

Analyze data relationships across rows without complex joins

The Core Pattern

SELECT *,
  LAG|LEAD|FIRST_VALUE|LAST_VALUE(column) OVER ( PARTITION BY col ORDER BY col ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING )
FROM table;
Section 01
LEAD & LAG: Month-over-Month
LAG retrieves a value from a previous row in the result set. LEAD retrieves a value from the next row. Together they enable row-to-row comparisons without self-joins.
SQL
-- Month-over-Month Sales Change
SELECT
  OrderMonth,
  CurrentMonthSales,
  LAG(CurrentMonthSales) OVER (ORDER BY OrderMonth) AS PreviousMonth,
  CurrentMonthSales - LAG(...) AS MoM_Change
FROM MonthlySales;

Section 02
LEAD & LAG: Customer Loyalty
Partition-based windows segment data by category (e.g., per customer). LEAD with PARTITION BY finds the next order within each customer's timeline, enabling cross-row analysis on grouped subsets.
SQL
-- Customer Order Intervals
SELECT
  CustomerID,
  OrderDate,
  LEAD(OrderDate) OVER (
    PARTITION BY CustomerID
    ORDER BY OrderDate
  ) AS NextOrder,
  DATEDIFF(day, OrderDate, LEAD(...)) AS DaysBetween
FROM Orders;

Section 03
FIRST_VALUE: Product Price Baselines
FIRST_VALUE returns the first value in the window according to the ORDER BY clause. It's perfect for comparing each row against a baseline (lowest price, first date, etc.).
SQL
-- Sales vs Lowest Price Per Product
SELECT
  ProductID,
  Sales,
  FIRST_VALUE(Sales) OVER (
    PARTITION BY ProductID
    ORDER BY Sales
  ) AS LowestPrice
FROM Orders;

Section 04
LAST_VALUE: Product Price Ceiling
LAST_VALUE returns the last value in the window. To include all rows, use a ROWS frame: ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING. This captures the maximum/final value across the entire partition.
SQL
-- Sales vs Highest Price Per Product
SELECT
  ProductID,
  Sales,
  LAST_VALUE(Sales) OVER (
    PARTITION BY ProductID
    ORDER BY Sales
    ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING
  ) AS HighestPrice
FROM Orders;

01 Categorize Data CASE + GROUP BY
CASE — Buckets Sales Tier Classification Categorize
CASE evaluates conditions top-to-bottom and returns the value for the first WHEN that is true. Wrapping it in a subquery lets you GROUP BY the computed label — turning raw numbers into named buckets. Change the thresholds below to see buckets shift live.
CASE WHEN <cond> THEN <val> ... ELSE <val> END
High > Med >
Add sale
SELECT Category, SUM(Sales) AS TotalSales
FROM (
 SELECT OrderID, Sales,
  CASE
   WHEN Sales > 50 THEN 'High'
   WHEN Sales > 20 THEN 'Medium'
   ELSE 'Low'
  END AS Category
 FROM Sales.Orders
) AS t
GROUP BY Category
ORDER BY TotalSales DESC;
02 Mapping Value → Label translation
CASE — Mapping Country Code Lookup Mapping
CASE as a mapping table lets you translate one set of values into another inline — no JOIN required. Each WHEN Country = … THEN … branch acts like a dictionary entry. The ELSE handles unmapped values gracefully.
CASE WHEN col = 'A' THEN 'x' WHEN col = 'B' THEN 'y' ELSE 'n/a' END

   ELSE 'n/a'
  END AS CountryAbbr

03 Quick Form Syntax Simple CASE vs Searched CASE
CASE — Quick Form Simple vs Searched Syntax Quick Form
SQL has two CASE syntaxes. The Searched CASE evaluates full boolean expressions in each WHEN — it is the most flexible form. The Simple CASE (Quick Form) places the column once after CASE and each WHEN just provides a match value. Both produce identical results for equality checks — the simple form is more concise.
CASE col WHEN 'val' THEN … END  vs  CASE WHEN col = 'val' THEN … END

04 Handling Nulls NULL-safe calculations
CASE — Nulls NULL to Zero Substitution Handling Nulls
NULL values poison aggregations — AVG() ignores NULLs, producing a misleadingly high average. Using CASE to replace NULL with 0 before aggregating gives a true average across all customers. This pattern is useful anywhere you want to count non-events as zero rather than absent.
CASE WHEN col IS NULL THEN 0 ELSE col END
Edit scores (leave blank = NULL)
AVG (nulls ignored): AVG (nulls=0):
SELECT
  CustomerID, Score,
  CASE
   WHEN Score IS NULL THEN 0
   ELSE Score
  END AS ScoreClean,
  AVG(CASE WHEN Score IS NULL
   THEN 0 ELSE Score END)
  OVER() AS AvgClean,
  AVG(Score) OVER() AS AvgRaw
FROM Sales.Customers;

05 Conditional Aggregation SUM(CASE…) pattern
CASE — SUM Count Filtered Orders per Customer Cond. Aggregation
Conditional aggregation uses SUM(CASE WHEN … THEN 1 ELSE 0 END) to count only rows that meet a condition — without a WHERE clause that would remove all other rows. This lets you count multiple different subsets in a single query pass, unlike using multiple sub-selects.
SUM(CASE WHEN condition THEN 1 ELSE 0 END) AS ConditionalCount
High >
SELECT
  CustomerID,
  SUM(
   CASE
    WHEN Sales > 30 THEN 1
    ELSE 0
   END
  ) AS TotalHighSales,
  COUNT(*) AS TotalOrders
FROM Sales.Orders
GROUP BY CustomerID;

01
Page Address = FileID : PageNumber
1:101 → FileID=1, PageNumber=101 — இந்த number SQL Server எப்படி decide பண்றது?
ஒரு page address இப்படி இருக்கும்:
1 : 101
↑ FileID
எந்த .mdf file? ↑ PageNumber
அந்த file-ல் எந்த page?
FileID என்ன?
SQL Server database = ஒன்று அல்லது அதிகமான .mdf / .ndf files ஆல் ஆனது.

Primary file = FileID 1 (.mdf)
Secondary files = FileID 2, 3, ... (.ndf)

நம்ம example-ல் ஒரே file மட்டும் → எல்லாமே FileID = 1
PageNumber என்ன?
ஒரு file-ல் pages sequential ஆக numbered: 0, 1, 2, 3, 4 ...

Page 0 = file-ல் முதல் page (system page)
Page 1 = இரண்டாவது page
Page 101 = 102-வது page in the file

Page size = 8KB (8192 bytes)
Page 101 starts at: 101 × 8192 = 827,392th byte

PageNumber 101-க்கான byte offset = 101 × 8,192 = 827,392 — SQL Server இந்த exact byte position-க்கு jump பண்ணும்!
💡 Key insight: 1:101 என்பது "File 1-ல் 102-வது page (0 indexed)". Page number directly physical file-ல் எந்த position-ல் page இருக்கும் என்று சொல்லும். SQL Server-க்கு separate address book தேவையில்லை — PageNumber × 8192 = exact byte position!
02
ஒரு .mdf File-ல் Pages எப்படி arranged?
Page 0 → Page 1 → Page 2 → ... sequential — system pages எங்கே, data pages எங்கே
Database file = pages-இன் array. Index = page number. Simple!
ஆனா எல்லா pages-உம் data pages அல்ல — சில system pages (SQL Server internal management).
Page 0 — File Header
Database பத்தின metadata.
Creation date, version, file size.
எல்லா file-லயும் page 0 = header.
Page 1 — PFS Page
Page Free Space tracker.
எந்த pages free/used/full என்று bitmap.
ஒரு PFS page = 8088 pages track பண்ணும்.
Page 2 — GAM Page
Global Allocation Map.
Extent-கள் free-ஆ mixed-ஆ என்று track.
1 bit = 1 extent (8 pages).

⚠️ Page numbers SQL Server decide பண்றது — நாம் control பண்ண முடியாது!
SQL Server free pages-ஐ find பண்ணி, அந்த page number-ஐ allocate பண்ணும்.
அதனால் Employees-க்கு 101, 102... Departments-க்கு 201, 202... வருது — இது real values இல்லை, SQL Server decide பண்ணும்.
03
Extent = 8 Pages = 64KB
SQL Server pages-ஐ ஒவ்வொன்னா allocate பண்றதில்லை — Extents-ஆ allocate பண்ணும்
SQL Server-ல் disk allocation unit = Extent.
1 Extent = 8 consecutive pages = 8 × 8KB = 64KB.
Table-க்கு space வேண்டும்போது, SQL Server ஒரு Extent-ஐ reserve பண்ணும்.

04
Table CREATE → INSERT → Pages Allocated
நாம் CREATE TABLE பண்றோம் → SQL Server எப்படி pages allocate பண்றது — step by step
1
CREATE TABLE dbo.Employees (...)
SQL Server system tables-ல் metadata store பண்ணும் (sys.objects, sys.columns).
இந்த step-ல் data pages allocate ஆவதில்லை! Table structure மட்டும் register ஆகும்.
2
First INSERT INTO dbo.Employees VALUES (...)
SQL Server GAM page scan பண்ணி free mixed extent தேடும்.
Free extent கிடைத்தால் → அந்த extent-ல் ஒரு page allocate பண்ணும்.
Page number = free extent-ல் உள்ள page-ன் sequential number in the file.
3
Page நிரம்ப → Next page allocate
ஒரு page full (8KB) ஆனால் → அடுத்த free page allocate ஆகும்.
Small table: mixed extent-ல் next free page (not necessarily consecutive).
Large table (>8 pages): uniform extent — 8 consecutive pages ஒரேயடியா கிடைக்கும்.
4
IAM Page — Table's page directory
SQL Server ஒவ்வொரு table-க்கும் IAM (Index Allocation Map) page maintain பண்ணும்.
IAM = "இந்த table-க்கு எந்த எந்த extents/pages belong பண்றன" என்ற map.
Table scan பண்ணும்போது, SQL Server IAM பார்த்து எந்த pages படிக்கணும் என்று decide பண்ணும்.

05
1:101 and 1:201 — இந்த exact numbers ஏன் வந்தது?
Employees-க்கு 101, Departments-க்கு 201 — இது fixed இல்லை, SQL Server freely choose பண்ணும்
⚠️ முக்கியமான point: 1:101, 1:201 என்ற exact numbers example-க்காக நாம் use பண்ணோம். Real-world-ல் SQL Server எந்த free page கிடைக்குதோ அதை allocate பண்ணும் — numbers predict பண்ண முடியாது!
🔴 Employees → Pages 1:101 to 1:105 — ஏன்?
1. Database create ஆனபோது, pages 0–99 system காரணங்களுக்கு (system tables, metadata, PFS, GAM, SGAM pages) ஆனது.
2. Employees table first row insert → SQL Server free page தேடுது → page 101 கிடைத்தது (page 100 = another system thing).
3. Row 11 → page full → page 102 allocate.
4. இப்படி consecutive ஆக 101, 102, 103, 104, 105 வந்தது.
🟠 Departments → Pages 1:201 to 1:202 — ஏன் 201?
1. Employees table-க்கு pages 101–105 போனதால், 106–199 range other objects-க்கு போயிருக்கலாம் (indexes, other tables).
2. Departments create பண்ணி first insert → next free page = page 201.
3. இது coincidence அல்ல, not fixed — SQL Server-ல் run பண்ணும்போது வேற numbers வரலாம்.
🔍 ACTUAL PAGE NUMBERS பார்க்க — SQL Server commands:
-- Table-ன் actual pages பார்க்க:
SELECT extent_page_id, allocated_page_page_id, page_type_desc,
       allocated_page_file_id
FROM sys.dm_db_database_page_allocations(
       DB_ID(), OBJECT_ID('dbo.Employees'), NULL, NULL, 'DETAILED')
WHERE page_type_desc = 'DATA_PAGE';

-- Output sample (இது YOUR database-ல் வேற numbers வரும்):
-- allocated_page_file_id | allocated_page_page_id | page_type_desc
-- 1 | 101 | DATA_PAGE
-- 1 | 102 | DATA_PAGE
-- 1 | 103 | DATA_PAGE

✅ Bottom line:
• FileID = எந்த .mdf file-ல் இருக்கு (usually 1)
• PageNumber = SQL Server free page கண்டுபிடிச்சு sequential number குடுக்கும்
• 101, 102... → Employees uniform extent (consecutive pages)
• 201, 202... → Departments different extent (gap ஏன்னா pages 106-200 other things-க்கு போயிருக்கு)
• இந்த numbers நாம் control பண்ண முடியாது — SQL Server Storage Engine decide பண்ணும்!

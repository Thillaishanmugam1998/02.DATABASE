01 Core Concepts

📄 Page

The smallest unit of data storage in SQL Server. Each page is exactly 8 KB. SQL Server reads and writes entire pages — never partial ones.

🗂️ Data Page

Stores actual row data from your tables. Contains a header (96 bytes), rows, free space, and an offset array at the bottom.

🔍 Index Page

Stores index entries — pointers that help SQL Server jump directly to the right data page instead of scanning every page.

💾 Data File (.mdf)

The main file on disk where SQL Server stores all pages — data pages + index pages. One .mdf file per database (primary).

🏔️ Heap

A table stored WITHOUT a Clustered Index. Rows are inserted in any order, scattered across pages. Fast writes, slow targeted reads.

🔎 Full Table Scan

When there is no usable index, SQL Server reads every page, every row from start to finish until the match is found.

Key Insight: SQL Server always reads/writes in full 8 KB pages. Even if you want 1 row, the entire page is loaded into memory (Buffer Pool). The page number format is FileID:PageID — e.g., 1:100 means File 1, Page 100.

🔵 Header (96 bytes)

Contains metadata: page number, object ID it belongs to, pointers to prev/next page in linked list, free space available, number of rows on the page.

🟡 Row Data

Each row is stored sequentially. Rows grow top → down. Every row has a small header + actual column data packed together.

⚪ Free Space

The gap between rows and offset array. New rows fill this space. When a page is full, SQL Server allocates a new page.

🔴 Offset Array (Slot Array)

Grows from bottom → up. Each slot stores the byte offset of a row. This lets SQL Server jump directly to any row position without scanning row-by-row within a page.
